## 第 1 章：认知基建 —— 我们在对付什么级别的怪物？

在正式开工前，我们先建立两个常识，这决定了我们后续为什么大费周章去搞配置，而不是直接暴力复制粘贴。

### 1.1 什么是 Google BigQuery？
你的组长给你下了个任务，让你从 BigQuery 导数据。你可能连发音都念不准，没关系。
你可以把 BigQuery 想象成一个**无边无际、拥有超人算力的“超级云端硬盘+超级Excel”综合体**。
普通的 Excel 处理几十万行就会卡得风扇狂转，但在 BigQuery 里，你可以存进**上千亿行**的数据。当你在里面点击“查询”时，Google 会偷偷调动几千台服务器帮你一瞬间算完。这就是为什么现代公司都把核心业务数据存进 BigQuery 的原因。

### 1.2 Excel 的物理与生理极限（阿喀琉斯之踵）
这是你要用生命记住的数字：**1,048,576**。
这是一个标准 Excel 工作表从上到下的极限行数。如果你硬要把 BigQuery 里的大于 104 万行的数据导出成 CSV，再强行用 Excel 打开：
* 你的电脑内存瞬间爆满。
* 你的 Excel 会出现卡死不动的白屏状态。
* 就算奇迹般打开了，第 1,048,577 行以后的所有数据都会**被无情地彻底截断丢失**，这就是致命的业务事故！

**核心思想：** 对于海量数据，永远不要全量导入。你需要运用这本手册里的方案，学会**“在云端过滤瘦身，只把精华提取到本地 Excel”**。

## 第 2 章：财产护卫 —— 别让信用卡意外出血

这是许多小白不敢注册云服务器的原因：点错一个按钮，次月账单爆降几千美金。别慌，我们来看看真实的计费规则。

### 2.1 BigQuery 的收费套路与免费额度
BigQuery 遵循的是“按量收费（Pay-as-you-go）”。就像水表跑多少算多少钱。但好消息是，Google 老板非常大方，每个月会给你送巨额保底配额：
- **存数据**：每月前 **10 GB** 免费。
- **查数据**：每月前 **1 TB（约等于 1024 GB）** 免费。
这个试用额度对于普通的临时提取统计任务来说，简直宽裕得过分。

### 2.2 终极白嫖法：零风险沙盒模式 (Sandbox)
如果你没有外币信用卡（Visa/Mastercard），或者是强迫症晚期害怕意外扣费，你可以启用**沙盒模式**。
**独门秘籍：** 只要你确保当前的 Google 账号（Gmail）没有绑定过任何付款方式，你直接在浏览器打开这个神奇的专属网址：
👉 `https://console.cloud.google.com/bigquery`
当你进去发现页面最上方有一条蓝色的横带，写着**“正在 Sandbox (沙盒) 模式下运行”**时，证明你成功了。
*代价：你在沙盒里自己建的数据表只能存在 60 天。但这不影响你今天去拉取你们团队长久存放的核心数据表。*

## 第 3 章：磨刀不误砍柴工 —— 环境探路与神器下载

你的电脑原本是个土包子，听不懂 Google 的高端语言，我们需要给它装一个翻译插头（专业名字叫 ODBC 驱动）。这是连接两个世界唯一的桥梁。

### 3.1 认清你的 Excel 基因（32 位还是 64 位）
乱装版本会导致桥梁坍塌。花 10 秒钟确认：
1. 双击桌面上的 Excel 图标，随便新建一个空白文档。
2. 眼睛往屏幕左上角看，点击最边上的 **【文件 (File)】**。
3. 顺着最左边灰色的栏位看到最底端，点击 **【账户 (Account)】**。
4. 屏幕中间右侧会出现一个带蓝色问号 ❓ 的按钮，写着 **【关于 Excel (About Excel)】**，点它！
5. 在弹出的版权窗口里，最上方第一行加粗字体的非常末尾，会有括号写着 `(64 位)` 或 `(32 位)`。
**记住并关闭软件。90% 以后的现代电脑都是 64 位的。**

### 3.2 寻宝之旅：获取官方 ODBC 驱动程序
1. 打开浏览器（强烈建议用 Google Chrome）。
2. 在地址栏输入或者直接去百度/谷歌搜索：`Google BigQuery ODBC Driver Download`
3. 或者是直接访问官方的这个隐蔽的窝点：
   `https://cloud.google.com/bigquery/docs/reference/odbc-jdbc-drivers`
4. 页面往下翻，看到标题是 **ODBC** 的地方。如果你上面查出自己是 64位，就毫不犹豫地点击 `下载 64位 .msi 文件`。

### 3.3 傻瓜式安装步骤深度解析
下载完毕后，你会得到一个看起来很像安装程序的图标。
1. 双击它。如果弹出系统安全警告，点击【是】或【允许运行】。
2. 映入眼帘的安装向导全篇英文，别怕，你只需要反复点击 **`[Next]` (下一步)**。
3. 遇到一个写着 `I accept the terms...`（我同意服务条款）的勾选框，**打上勾**。
4. 继续点 **`[Next]`** -> 最后点 **`[Install]` (安装)**。
5. 进度条跑满后，点击 **`[Finish]` (完成)**。
恭喜，第一道难关已过！

## 第 4 章：架构师附体 —— 配置系统底层连接通道 (DSN)

到了最“极客”的环节！我们要像一个真正的黑客一样，在 Windows 的核心深处接一根管子，这根管子直通 Google。

### 4.1 唤醒系统隐藏组件 ODBC
1. 看着你电脑屏幕的左下角，看到那个代表 Windows 的田字格徽标了吗？**点它打开开始菜单。**
2. **手离开鼠标，不要管搜索框在哪，用手摸键盘直接盲打：** `O-D-B-C`
3. 这时，开始菜单会神奇地浮现出最佳匹配项，叫 **【ODBC 数据源 (64 位)】**。（如果你刚才装的 32 位就选 32 的那个）。它可能有一个齿轮带两个小罐子的图标。
4. 单击打开它。这就是我们的指挥中心。

### 4.2 建立专属你的高维通道
1. 窗口的最上方有一排标签卡（像抽屉一样），点击第二个抽屉，名字叫 **【系统 DSN (System DSN)】**。
2. 放眼望向右侧中间，点击那个名叫 **【添加 (Add...)】** 的按钮。
3. 弹出一个长长的列表，里面都是各种名号的“驱动”。滚动鼠标滚轮一直往下到底。
4. 寻找一个名字以 **【Simba ODBC Driver for Google BigQuery】** 开头的家伙，用你的鼠标左键盘选中它（它会变蓝）。
5. 点击右下角的 **【完成 (Finish)】**。

### 4.3 黑客时刻：获取 OAuth 授权密钥 (Token)
重头戏来了，接下来弹出的框框虽然密密麻麻，但我们只攻破三点。

**第一点：Data Source Name (起个花名)**
在它旁边空白的长条格子里，敲入你喜欢的字母，它是这根连接管子的代号。比如输入：`CompanyData`。

**第二点：OAuth Mechanism (开门方式)**
在它右边的下拉框里，默认是 `Service Authentication`。点它，换成向下滚一点的 **【User Authentication】**（意思是用你的本人账号认证）。

**第三点：获取终极大门钥匙 (Refresh Token)**
就在下拉框的下方，有一个按钮，上面写着 **【Sign In...】**。
1. **深吸一口气，点击它。**
2. 你的电脑会突然自动弹出一个网页（浏览器）。
3. 这个网页你极其眼熟，“正在使用 Google 账号登录”。**用你们长官赋予了权限的那个 Google 邮箱登录。**
4. 网页跳转，出现警告：“Simba 需要访问您的 BigQuery 数据...”。别犹豫，猛击蓝色的 **【允许 (Allow)】**。
5. 跳转后，网页中央会出现一行长得极其离谱的乱码，下面还有个图标叫你复制它。
6. **全选并右键复制那段天文乱码！**
7. 缩小浏览器，回到那个恐怖的 ODBC 英文设置框。
8. 找到中间偏下一个长长的格子，叫 **【Refresh Token】**。把刚才复制的乱码，强行 **【粘贴 (Ctrl+V)】** 进去。

*(注：部分最新版的插头会自动写入这段乱码，如果你点完允许回来发现格子里已经填有乱码了，这一步可省略。)*

**第四点：选择管辖地 (Catalog/Project)**
再往下看，找到写着 `Catalog (Project)` 的下拉框，点击箭头展开。如果在之前的账户操作准确无误，下拉菜单里应该出现你们公司那个“金贵”的数据库项目名称。选中它。

### 4.4 点火测试：连接成功的终极信号
一切配置完毕，向你的右下方看。
找到一个孤零零的按钮 **【Test... (测试)】**。
点击它，屏息以待 2 秒钟。如果弹出一个极其短小的提示框，上面写着纯净无暇的一个英文单词：
🚀 **SUCCESS (成功)** 🚀

那么，起立为自己鼓掌三秒钟。你做到了 90% 的人卡住的地方。
点击测试框的 **【OK】**，继续点击设置框的 **【OK】**，最后由于没用了，点击 ODBC 主界面的 **【确定】**。清理战场结束。

## 第 5 章：见证奇迹 —— 在 Excel 中降维打击海量数据

一切基建已经结束，接下来的步骤将成为你在职场“摸鱼”或者闪耀全场的日常手法。

### 5.1 召唤古老魔法：从获取数据开始
1. 打开 Excel。这是日常你最熟悉的软件。
2. 保持冷静新建一个完全空白的工作表。
3. 点击顶端的选项卡大栏目：**【数据 (Data)】**。
4. 放眼到最最左边的第一个大图标，通常叫做 **【获取数据 (Get Data)】**，点它。
5. 沿着级联菜单往下找：选择 **【来自其他源 (From Other Sources)】**。
6. 最后点击里面最不显眼的那一项：**【来自 ODBC (From ODBC)】**。

### 5.2 漫游数据仓库：导航器的使用
此时，屏幕中央会端庄地弹出一个极简的小窗。
1. `数据源名称 (DSN)`：点击它的下拉箭头。
2. 你眼前一亮，那个你亲手起的名字，比如 `CompanyData`，赫然在列！选中它！
3. 然后什么高级选项也不用管，直接点击 **【确定 (OK)】**。

*(插曲应对：如果在此刻或者下一步，Excel 突然像发疯一样弹窗，让你填写“凭据”或者“数据库账号密码”，这非常正常！不用慌，将眼光移向左边，点击 **【默认或自定义 (Default or Custom)】**，然后保证用户名密码都是彻头彻底的绝对空白，再次点击下面低调的【连接 (Connect)】。不用怀疑，直接空着！)*

接下来，**【导航器】** 窗口出现。它的左边就像你查看 C盘 D盘的文件夹一样。
一层一层点击带加号的层级结构（比如点击项目名，然后出现数据库集，然后找到你要的那张具体表），当鼠标停留在最底层的表名时，右边的巨大空白区域会加载几秒钟，然后展现出“预览数据”。
这里能确认这是不是你想要的。

**⚠️ 高危警戒区 (DO NOT CLICK)：**
在导航器的正右下方，有一个极其诱人的普通按钮叫做 **【加载 (Load)】**。
如果你心里知道这张表极有可能破了一百万行，**绝对不要点它！绝对不要碰它！** （点完必定 Excel 重启报废）。

我们需要把目光向右平移 2 厘米。点击那个叫做 **【转换数据 (Transform Data)】**（有的老版本叫【编辑】）的按钮。

### 5.3 数据斩舰刀：Power Query 终极瘦身法（最重要！）
你点击以后，Excel 会变成像异次元时空一样，跳出一个土黄色的、功能比普通 Excel 庞杂一百倍的全新窗口。这叫 **Power Query 编辑器**。
这也是微软花重金打造的数据清理超级武器，不要怕它。你在里面做的所有删除、过滤操作，都不会“消灭”存在 Google 里的云端原始数据，只会改变最终流向你本地表格的模样。

**操作示范 A：精简列（减负）**
很多原始大数据表会带有几百个字段（列名）。你并不关心“用户登录设备来源 ID 哈希值”，你只关心“收入金额”。
- 找到界面上的数据预览表头。
- 用鼠标点击你需要扔掉的列的名字首，选中它。
- 右键单击那一列表头，选择 **【删除 (Remove)】**。它就消失了！极大减轻数据重量！
- *提示：如果删错了，你可以看这个窗口的最右边，有个叫“应用的步骤”的菜单，里面有你每一步历史。鼠标指过去出现一个红色的小红叉 X，点击红叉就能彻彻底底地撤销！*

**操作示范 B：时间与切片筛选（腰斩海量行数）**
这是你避免百万行崩溃的最核心招式。比如那表有十年数据，共 5000 万行。你需要 2024 年第一季度的。
- 找到能代表时间或者分类的那一列表头。
- 表头的名字最右边有个很小的三角形 🔻。点开它。
- 界面会有点像 Excel 原生筛选——勾选你要的内容（比如勾选特定的几个城市名，或者通过“日期筛选器”选大于特定日期）。
- 筛选后，所有不符合条件的数据直接被你“腰斩屏蔽”。（这时候其实就是让云端的服务器帮你把千万行数据过滤成几万行精华）。

### 5.4 关门大吉：闭合并倾泻数据
当你利用精简列和过滤筛选确信：“现在留下来的结构，顶多只有几万行或十万行”之后。
把你的鼠标移动到这个庞杂黄色大窗口的**极度左上角**。
这儿有一个画着类似抽屉或云盘的特大号图标，名叫：
**【关闭并加载 (Close & Load)】**
——毫不犹豫地点下去！

这个异次元窗口此时瞬间坍塌消失。你会回到最熟悉的那个空白 Excel 里。
看屏幕最右侧，出现了一个进度条提示“正在提取数据...正在加载...”。几秒或几十秒后（完全取决于你过滤剩下之后的数据重量和你的网速）。

数据被无比整齐、带有漂亮绿色表头的主题颜色表格的形式，完美导入到了你的工作表内！

**此时此刻，你作为零编程基础的小白，完成了那些要学半年 Python Pandas 的数据人才能做到的事情！**

## 第 6 章：进阶与自动化 —— 以后该如何偷懒？

只要你完成了第五章并且把这个包含数据的 Excel 文件**保存**了（比如存在桌面，取名为 `每月业绩汇总.xlsx`），下次你需要再干活，就是分分钟的事情。

### 6.1 如何做到每天只按一个按钮就完成工作？
下周，小组里问你：“喂，数据云端有上周的入库了，帮我更新一下本地 Excel 汇报表格呗？”

1. 打开你之前存好的 `每月业绩汇总.xlsx`。
2. 保持淡定，左手端起茶杯。
3. 把鼠标随意点中带颜色的数据区域里面任意一个格子。
4. 顶部菜单会出现隐藏标签 **【表格设计/查询设计】**，但你不用找那个。
5. 你直接找到顶部菜单栏的主页上的 **【数据 (Data)】**。
6. 点击那个带个刷新大图标的 **【全部刷新 (Refresh All)】**。

接下来是魔法时间。Excel 底层已经记住了你上一次连接 Google 和用 Power Query“删列、筛选”的全套动作（它像一个忠实的机器人录像机一样背了下来）。
它会自己连接那个管子，到 BigQuery 爬去最新更新好的对应条件的数据，覆盖到现在的电脑屏上。

### 6.2 面对云端经常添加和改版，该怎么处理？
只要你之前不胡乱用 Power Query 写高级函数，只用的左点右点。如果今天 BigQuery 云端被开发人员加了三个大新字段，你点了**全部刷新**之后，这三个崭新的字段会顺滑地作为新增列出现在你的表格最右边。这就是 ODBC 管线的强大兼容优势。

## 第 7 章：急诊抢救室 —— 新手必踩坑与常见报错解析大全

哪怕是保姆教程，在千万台不同的电脑环境下，偶有妖风作怪。这份排错清单囊括了 99% 的错误根源。

#### ❌ 症状 1：安装驱动时报错无法继续
- **诊断**：你下载了 `.msi` 后果断双击，却报错 “需要更高权限” 或者类似的话。
- **解药**：公司的电脑可能上了内网安全狗。深呼吸，这不归你管，礼貌地联系你们公司网管 IT 部门输入一下管理员密码安装即可。

#### ❌ 症状 2：ODBC 配置 OAuth 测试一直弹 Error Failed
- **诊断**：Token 获取失败或者网络问题。
- **解药 1**：检查你电脑当前的 Wi-Fi 网络能不能顺畅访问境外网络（比如 Google 本身有没有被屏蔽防火墙墙掉？），如果企业内网本身就没打通对外部海外云计算的代理，则插头无法伸过去。由于是云服务，首要保证全程顺网。
- **解药 2**：回忆下当时获取 token 那长长一串乱码，是不是复制漏了最后一位或者前后不小心带上了空格？一定要严丝合缝。

#### ❌ 症状 3：在 Excel 点击获取 ODBC 数据时弹窗要求提供数据库账户密码
- **诊断**：这个几乎每个人都会遇到，这是 Excel 自作聪明的设计。
- **解药**：仔细看我第五章提到的高深插曲。不管它，点击它的左侧 `默认/通用 (Default/Custom)` 选项卡。绝对不在那两个密码文本框里敲任何字母字。不填即是真理，直接点确认连接。

#### ❌ 症状 4：不慎点击了“加载”，数据像泥石流一样淹没了你的内存
- **解药**：如果电脑疯狂转圈死机。长按你电脑电源键重启（或者按 `Ctrl+Shift+Esc` 呼出任务管理器强行终结 Microsoft Excel 进程）。如果你下次还要操作一次，万分警惕，当导航器出来时一定要点右边灰灰的【转换数据（Transform Data）】，别手快！
